# 3DGS 点云处理工具 (3dgs-pp) 产品需求文档

## 1. 项目概述

### 1.1 项目名称
3DGS Point Cloud Processing Tool (3dgs-pp)

### 1.2 项目背景
3D Gaussian Splatting (3DGS) 是一种新型的三维重建技术，其生成的点云数据通常存储在 PLY 文件中。随着场景规模增大，PLY 文件的体积也会急剧增长，给数据查看、处理和传输带来挑战。本工具旨在提供高效的 3DGS PLY 文件处理能力，支持懒加载、终端浏览和空间分块功能。

### 1.3 项目目标
- 提供高效的大型 PLY 文件懒加载机制
- 支持终端交互式浏览高斯点数据
- 实现灵活的空间分块导出功能
- 支持按条件过滤高斯椭球，清理异常或不需要数据
- 支持高斯椭球下采样，减少模型复杂度

## 2. 功能需求

### 2.1 懒加载 PLY 文件读取

**功能描述**：支持读取 3DGS 格式的 PLY 文件，采用懒加载方式，避免一次性加载全部数据到内存。

**详细需求**：
- 支持标准 3DGS PLY 文件格式解析
- 仅在需要时加载具体高斯点数据
- 支持文件元数据（顶点数量、属性列表等）的快速读取
- 内存占用控制：元数据加载后内存占用 < 10MB
- 支持超大文件（> 10GB）的处理

**PLY 属性列表**（3DGS 标准）：
- x, y, z：位置坐标
- f_dc_0, f_dc_1, f_dc_2：球面调和基直流分量
- f_rest_0 到 f_rest_44：球面调和基剩余分量（45个）
- opacity：不透明度
- scale_0, scale_1, scale_2：缩放
- rot_0, rot_1, rot_2, rot_3：旋转（四元数）

### 2.2 终端列表展示与翻页

**功能描述**：在终端中以列表形式展示高斯点数据，支持上下翻页浏览。

**详细需求**：

#### 2.2.1 展示模式
- 默认展示核心属性：索引、x/y/z 坐标、opacity
- 支持展开查看更多属性（f_dc、scale、rot）
- 支持自定义选择展示的属性列
- 数值格式化：浮点数保留 6 位小数

**展开模式显示属性：**
- 索引、x、y、z、opacity
- f_dc_0, f_dc_1, f_dc_2
- scale_0, scale_1, scale_2
- rot_0, rot_1, rot_2, rot_3

#### 2.2.2 交互控制
- `j` / `↓`：向下翻一页
- `k` / `↑`：向上翻一页
- `g` / `Home`：跳转到第一页
- `G` / `End`：跳转到最后一页
- `:` / `number` + `Enter`：跳转到指定索引
- `/` + `关键词`：搜索包含特定值的点
- `e`：展开/折叠详细属性
- `q`：退出浏览模式
- `?`：显示帮助信息

#### 2.2.3 页面配置
- 每页默认显示 20 条记录
- 支持通过命令行参数自定义每页条数
- 显示当前位置：`第 X 页 / 共 Y 页 (记录 Z1-Z2 / 共 N)`

### 2.3 空间分块与导出

**功能描述**：按照指定的 X/Y/Z 方向分块数量，将点云分割成多个块，分别导出为 PLY 文件。

**详细需求**：

#### 2.3.1 分块策略
- 输入格式：`Nx*Ny*Nz`，例如 `2*3*2`
- X 方向分 Nx 块，Y 方向分 Ny 块，Z 方向分 Nz 块
- 分块边界：基于点云的包围盒（AABB）均匀划分
- 边界点处理：位于分块边界上的点归属于索引较小的块

#### 2.3.2 导出文件
- 文件命名格式：`{原文件名}_block_{i}_{j}_{k}.ply`
- 每个分块文件包含完整的 PLY 头部信息
- 保留所有原始属性数据
- 生成 `{原文件名}_block_info.json`，记录各分块的：
  - 空间范围（min/max x/y/z）
  - 点数量
  - 文件名映射

#### 2.3.3 进度反馈
- 显示整体进度百分比
- 显示已处理/总点数
- 显示已生成/总分块文件数
- 估算剩余时间

### 2.4 空间统计概览

**功能描述**：对 PLY 文件中指定参数进行统计分析，展示最大值、最小值、均值、中值、四分位数以及 5% 10% 20% 50% 90% 95% 分位数，支持快捷键切换查看不同统计维度。

**详细需求**：

#### 2.4.1 统计维度

支持对所有 PLY 数值属性进行统计，包括但不限于：

- **空间坐标**：x, y, z
- **外观属性**：opacity
- **球面调和系数**：f_dc_0, f_dc_1, f_dc_2, f_rest_0 ~ f_rest_44
- **几何属性**：scale_0, scale_1, scale_2, rot_0, rot_1, rot_2, rot_3

#### 2.4.2 统计指标

对选定属性计算以下统计值：

| 统计指标 | 说明 |

| 最大值 (Max) | 数据集中的最大值 |
| 最小值 (Min) | 数据集中的最小值 |
| 均值 (Mean) | 算术平均值 |
| 中值 (Median) | 50% 分位数 |
| Q1 (25% 分位数) | 第一四分位数 |
| Q2 (50% 分位数) | 第二四分位数（同中值） |
| Q3 (75% 分位数) | 第三四分位数 |
| 5% 分位数 | 下有 5% 数据的值 |
| 10% 分位数 | 下有 10% 数据的值 |
| 20% 分位数 | 下有 20% 数据的值 |
| 50% 分位数 | 下有 50% 数据的值（同中值） |
| 90% 分位数 | 下有 90% 数据的值 |
| 95% 分位数 | 下有 95% 数据的值 |
| 标准差 (Std) | 总体标准差 |
| 偏度 (Skewness) | 分布偏斜程度 |
| 峰度 (Kurtosis) | 分布峰态程度 |

#### 2.4.5 属性分布图绘制

**功能描述**：将指定参数的值分布绘制成图表并保存到指定目录，支持交互式查看和非交互式批量生成。

**详细需求**：

**图表类型：**
- **直方图 + KDE 曲线**：展示数据分布密度
- **箱线图 (Box Plot)**：展示中值、四分位数、异常值范围
- **小提琴图 (Violin Plot)**：结合直方图和箱线图，展示分布密度与四分位数

**图表元素：**
- 横轴：属性值
- 纵轴：频数 / 密度
- 在图上标注：均值（虚线）、中值（实线）、Q1/Q3（点划线）
- 标题包含：属性名、样本数
- 颜色主题：暗色背景（适合嵌入文档）

**交互触发：**
- `p`：在当前交互模式下，为当前属性绘制分布图并保存到默认目录
- `P`（Shift+p）：为所有核心属性（x, y, z, opacity, scale_0, scale_1, scale_2）批量绘制分布图

**命令行触发：**
```bash
# 为非交互模式绘制指定属性的分布图
3dgs-pp stat --attr opacity --plot <ply_file>

# 批量绘制所有核心属性
3dgs-pp stat --all --plot --output-dir ./plots <ply_file>

# 指定输出目录和图表类型
3dgs-pp stat --attr x --plot --output-dir ./charts --type box <ply_file>
```

**输出配置：**
- 默认输出目录：`{原文件名}_stats_plots/`，与原始 PLY 文件同目录
- 支持 `--output-dir DIR` 覆盖
- 文件命名格式：`{属性名}_distribution.png`
- 输出格式：PNG（默认，分辨率 300 DPI）
- 图表尺寸：800x500 像素（单属性），1200x800 像素（多属性对比）

#### 2.4.6 性能要求

- 懒加载计算：统计值按需计算，不一次性加载全部数据
- 单次统计计算时间：< 5 秒（1 亿点文件，单属性）
- 内存占用：统计过程中峰值内存 < 300MB
- 分布图绘制时间：< 10 秒（1 亿点文件，单属性）

### 2.5 高斯椭球过滤

**功能描述**：根据指定参数的数值或百分比范围条件，过滤掉符合条件的高斯椭球，生成新的 PLY 文件。支持多个参数各自独立的过滤条件，也支持对多个参数统一应用相同的过滤操作。

**详细需求**：

#### 2.5.1 过滤条件类型

| 操作符 | 含义 | 示例 | 说明 |
|--------|------|------|------|
| `>` | 大于某数值 | `opacity>0.1` | 过滤掉 opacity > 0.1 的椭球 |
| `>=` | 大于等于某数值 | `scale_0>=0.5` | 过滤掉 scale_0 >= 0.5 的椭球 |
| `<` | 小于某数值 | `opacity<0.01` | 过滤掉 opacity < 0.01 的椭球 |
| `<=` | 小于等于某数值 | `z<=0` | 过滤掉 z <= 0 的椭球 |
| `==` | 等于某数值 | `rot_0==0` | 过滤掉 rot_0 == 0 的椭球 |
| `!=` | 不等于某数值 | `opacity!=0` | 过滤掉 opacity != 0 的椭球 |
| `>P` | 大于某百分比（分位数） | `opacity>P95` | 过滤掉 opacity > 95% 分位数的椭球 |
| `>=P` | 大于等于某百分比 | `scale_0>=P10` | 过滤掉 scale_0 >= 10% 分位数的椭球 |
| `<P` | 小于某百分比 | `opacity<P5` | 过滤掉 opacity < 5% 分位数的椭球 |
| `<=P` | 小于等于某百分比 | `z<=P50` | 过滤掉 z <= 50% 分位数的椭球 |
| `~` | 介于某数值范围（含边界） | `x~[-10,10]` | 过滤掉 x 在 [-10, 10] 范围内的椭球 |
| `!~` | 不介于某数值范围 | `x!~[-10,10]` | 过滤掉 x 不在 [-10, 10] 范围内的椭球 |
| `~P` | 介于某百分比范围（含边界） | `opacity~P[5,95]` | 过滤掉 opacity 在 5%~95% 分位数之间的椭球 |
| `!~P` | 不介于某百分比范围 | `z!~P[10,90]` | 过滤掉 z 不在 10%~90% 分位数之间的椭球 |

#### 2.5.1.1 间接参数（派生属性）

除了 PLY 文件中的原始属性，过滤还支持以下**间接参数**。这些参数根据高斯椭球的三个缩放值（`scale_0`, `scale_1`, `scale_2`）实时计算，无需存储到文件中。

> **计算基础**：3DGS 中的 `scale_0/1/2` 存储的是对数缩放值，实际半轴长度为 `exp(scale_0)`, `exp(scale_1)`, `exp(scale_2)`。为便于使用，间接参数对原始 scale 值排序后记为 `s1 <= s2 <= s3`（对应半轴 `l1 <= l2 <= l3`）。

| 间接参数名 | 计算方式 | 说明 |
|-----------|---------|------|
| `volume` | `exp(scale_0) * exp(scale_1) * exp(scale_2)` | 高斯椭球体积（成正比） |
| `longest_axis` | `max(exp(scale_0), exp(scale_1), exp(scale_2))` | 最长半轴长度 |
| `shortest_axis` | `min(exp(scale_0), exp(scale_1), exp(scale_2))` | 最短半轴长度 |
| `sphericity` | `l1 / l3`（排序后最短/最长半轴比） | 接近圆球程度：范围 [0,1]，1 为完美球体，越小越扁平/细长 |
| `disceness` | `l1 / l2`（排序后最短/中位半轴比） | 接近圆盘/椭圆饼程度：范围 [0,1]，接近 0 表示一个维度远小于另外两个，呈盘状 |
| `rodness` | `l2 / l3`（排序后中位/最长半轴比） | 接近棒/针状程度：范围 [0,1]，接近 0 表示两个维度远小于第三个，呈棒状 |

**使用示例**：

```bash
# 过滤掉体积极小的高斯椭球（体积 < 5% 分位数）
3dgs-pp filter --filter "volume<P5" scene.ply

# 过滤掉最长轴过大的异常椭球
3dgs-pp filter --filter "longest_axis>P95" scene.ply

# 过滤掉接近完美球体的椭球（sphericity > 0.9）
3dgs-pp filter --filter "sphericity>0.9" scene.ply

# 过滤掉呈扁平圆盘状的椭球（disceness < 0.1）
3dgs-pp filter --filter "disceness<0.1" scene.ply

# 过滤掉呈细长棒状的椭球（rodness < 0.1）
3dgs-pp filter --filter "rodness<0.1" scene.ply

# 组合：保留既不是球体也不是棒状的椭球
3dgs-pp filter --and --filter "sphericity>0.8" --filter "rodness>0.8" scene.ply

# 过滤掉极小且扁平的椭球（体积和圆盘程度同时满足）
3dgs-pp filter --and --filter "volume<P10" --filter "disceness<0.2" scene.ply
```

**与原始属性组合使用**：

间接参数可以与原始属性在过滤表达式中自由组合：

```bash
# 过滤掉不透明度高但体积极小的椭球（可能是噪点）
3dgs-pp filter --and --filter "opacity>0.5" --filter "volume<P5" scene.ply

# 过滤掉位置在远处且形状异常的椭球
3dgs-pp filter --and --filter "z>100" --filter "sphericity<0.1" scene.ply
```

**性能说明**：间接参数在计算时只需要读取 `scale_0, scale_1, scale_2` 三个原始属性，通过 numpy 向量化计算生成派生列，计算开销极低（< 10ms 对于 1 亿点）。

#### 2.5.2 多参数过滤

支持两种指定方式：

**方式一：独立条件（推荐）**

对每个参数单独指定过滤条件，通过多次 `--filter` 参数组合：

```bash
3dgs-pp filter --filter "opacity<P5" --filter "scale_0>P90" scene.ply
```

等价于：过滤掉 `(opacity < P5 分位数) OR (scale_0 > P90 分位数)` 的椭球

**方式二：统一操作**

对多个参数统一应用相同的比较操作：

```bash
3dgs-pp filter --op ">" --threshold 0.1 --attr opacity scale_0 scene.ply
```

等价于：过滤掉 `(opacity > 0.1) OR (scale_0 > 0.1)` 的椭球

#### 2.5.3 逻辑组合

- 多个 `--filter` 条件之间默认使用 **OR** 逻辑（满足任一条件即被过滤）
- 通过 `--and` 标志切换为 **AND** 逻辑（满足所有条件才被过滤）
- 通过 `--keep` 标志反转逻辑（保留符合条件的点，而非过滤掉）

```bash
# OR 逻辑（默认）：过滤掉 opacity<P5 或 scale_0>P90 的椭球
3dgs-pp filter --filter "opacity<P5" --filter "scale_0>P90" scene.ply

# AND 逻辑：仅过滤掉同时满足 opacity<P5 且 scale_0>P90 的椭球
3dgs-pp filter --and --filter "opacity<P5" --filter "scale_0>P90" scene.ply

# 保留模式：保留 opacity>0.1 的椭球（丢弃其余）
3dgs-pp filter --keep --filter "opacity>0.1" scene.ply
```

#### 2.5.4 交互模式

```bash
3dgs-pp filter --interactive scene.ply
```

**交互流程**：

1. 自动进入统计概览模式（复用 stat 功能的统计能力）
2. 用户浏览各属性统计值，确认过滤阈值
3. 输入过滤表达式，实时预览过滤结果（保留/丢弃的点数和比例）
4. 确认或调整过滤条件，支持多轮迭代
5. 最终确认后将结果写入新文件

**交互控制**：

- `a` / `d` 或 `←` / `→`：切换查看属性（复用 stat 功能）
- `f` + `回车`：输入过滤表达式，预览结果
- `F` + `回车`：追加过滤条件（多条件组合）
- `Enter`（在预览模式下）：确认过滤并写入文件
- `Esc`：取消当前过滤条件
- `o`：指定输出文件路径
- `q`：退出交互模式

**预览示例**：

```
过滤条件: opacity<P5

  将被过滤: 2,619,206 (5.00%)
  将保留:   49,764,923 (95.00%)
  估算输出文件大小: 1.82 GB
  预计处理时间: ~12 秒

  [Enter] 确认写入   [f] 修改条件   [Esc] 取消   [q] 退出
```

#### 2.5.5 输出结果

- 生成新的 PLY 文件，包含过滤后保留的高斯椭球
- 保留所有原始属性数据
- 在 PLY header 中添加注释记录过滤参数
- 默认输出文件命名：`{原文件名}_filtered.ply`
- 支持 `--output FILE` 指定输出路径
- 控制台输出统计信息：
  - 原始点数
  - 被过滤的点数和比例
  - 保留的点数和比例
  - 各条件命中的点数（多条件时）
  - 处理时间

#### 2.5.6 进度反馈

- 百分比分位数计算进度
- 过滤处理进度
- 已处理/总点数
- 估算剩余时间

#### 2.5.7 性能要求

- 分位数计算：按需缓存，同一属性多次使用相同分位数时不重复计算
- 过滤处理速度：> 200 万点/秒
- 内存占用：过滤过程中峰值 < 300MB

### 2.6 高斯椭球下采样

**功能描述**：对 3DGS 模型进行下采样，按照指定百分比或目标数量保留高斯椭球，生成新的 PLY 文件。

**详细需求**：

#### 2.6.1 采样策略

**V1.0 - 均匀采样（优先实现）**：
- 按索引均匀间隔采样：`保留点索引 = i * step`，其中 `step = 原始点数 / 目标点数`
- 简单高效，适合快速预览
- 适用于点分布相对均匀的场景

**可选高级策略（后续版本）**：

**策略 A - 基于不透明度的重要性采样**：
- 优先保留不透明度高的高斯椭球（对渲染质量影响更大）
- 可配置不透明度权重：`score = opacity * scale_factor`
- 适合需要保持视觉质量的场景

**策略 B - 基于体积的采样**：
- 根据高斯椭球的体积（scale 乘积）进行采样
- 优先保留体积大的高斯椭球
- 适合几何结构保持

**策略 C - 空间聚类采样（Voxel-based）**：
- 将空间划分为体素网格
- 每个体素保留一个代表性高斯椭球（不透明度最高或居中）
- 能够更好地保持空间分布

**策略 D - 随机采样**：
- 完全随机选择保留的点
- 可设置随机种子保证可复现性
- 适合数据增强或统计分析

**策略 E - 高斯椭球合并（Gaussian Merging）**：
- 按空间位置将临近的高斯椭球合并成一个新的高斯椭球
- 不是简单保留/删除，而是通过加权平均整合临近高斯的属性
- 能够更好地保持场景的整体视觉效果
- 适用于需要平滑下采样效果的场景

#### 2.6.2 输入参数

支持两种指定方式（二选一）：
- `--ratio R`：保留百分比，R 为 0 到 1 之间的小数（如 `--ratio 0.5` 表示保留 50%）
- `--count N`：保留的高斯椭球数量，N 为正整数

通用参数：
- `--output FILE`：输出文件路径（默认：`{原文件名}_downsampled.ply`）
- `--method NAME`：采样方法（默认：`uniform`）
  - `uniform`：均匀采样
  - `opacity`：基于不透明度采样（可选）
  - `voxel`：体素聚类采样（可选）
  - `random`：随机采样（可选）
  - `merge`：高斯椭球合并（可选）
- `--seed S`：随机种子（仅随机采样时使用）

#### 2.6.3 输出结果

- 生成新的 PLY 文件，包含下采样后的高斯椭球
- 保留所有原始属性数据
- 在 PLY header 中添加注释记录下采样参数
- 控制台输出统计信息：
  - 原始点数
  - 保留点数
  - 压缩比例
  - 处理时间

#### 2.6.4 进度反馈
- 显示采样进度
- 显示已处理/总点数
- 显示采样方法和参数

```

参数：
- `--filter "expr"`：过滤表达式，可多次指定
  - 数值比较：`attr>value`, `attr<=value`, `attr~[min,max]`, `attr!~[min,max]`
  - 百分比比较：`attr>P95`, `attr<=P5`, `attr~P[5,95]`, `attr!~P[10,90]`
- `--and`：多条件使用 AND 逻辑（默认 OR）

示例：

# 保留 opacity > P5 的椭球（丢弃其余）
3dgs-pp filter --keep --filter "opacity>P5" scene.ply

# 过滤掉 x 不在 [-50, 50] 范围内的椭球
3dgs-pp filter --filter "x!~[-50,50]" scene.ply

# 交互模式
3dgs-pp filter --interactive scene.ply
```

### 5.2 关键技术点

1. **懒加载实现**：基于文件偏移量索引，使用 `seek()` 按需读取
2. **终端 UI**：使用 `curses` 或 `rich` 库实现交互式界面
3. **进度条**：使用 `tqdm` 库展示处理进度
4. **统计分析**：使用 `numpy.percentile` 和 `numpy.mean/std` 高效计算，通过 `np.memmap` 避免全量加载
5. **过滤引擎**：解析过滤表达式，支持数值和百分比分位数条件，通过布尔掩码实现多条件 OR/AND 组合
6. **下采样算法**：
   - 均匀采样：索引步进算法
   - 重要性采样：基于不透明度/体积的权重排序
   - 体素采样：空间网格划分 + 局部最优选择

### 5.3 下采样算法说明

#### 均匀采样（Uniform Sampling）
**原理**：按固定间隔从原始点云中选取点
- 计算采样步长：`step = original_count / target_count`
- 选取索引：`i * step` 向下取整
- **优点**：O(N) 时间复杂度，实现简单，速度极快
- **缺点**：可能丢失重要特征，不适合点密度变化大的场景

#### 基于不透明度采样（Opacity-based Sampling）
**原理**：优先保留不透明度高的高斯椭球
- 计算每个点的重要性分数：`score = opacity`
- 按分数降序排序，选取前 N 个
- **优点**：保留视觉上更重要的点，渲染质量下降更平缓
- **缺点**：需要 O(N log N) 排序时间，可能破坏空间分布均匀性

#### 体素聚类采样（Voxel-based Sampling）
**原理**：空间划分后每个体素保留代表点
1. 根据点云边界和目标点数计算体素大小
2. 将每个点分配到对应体素
3. 每个体素中选择不透明度最高的点作为代表
- **优点**：保持空间分布均匀性，同时考虑重要性
- **缺点**：内存占用较高（需要构建体素索引），实现较复杂

#### 随机采样（Random Sampling）
**原理**：随机选择 N 个点保留
- 使用 Fisher-Yates shuffle 或 reservoir sampling
- 可设置随机种子保证可复现
- **优点**：无偏采样，适合统计分析
- **缺点**：可能丢失重要特征

#### 高斯椭球合并（Gaussian Merging）
**原理**：将空间临近的高斯椭球合并为一个新的高斯椭球
1. 使用体素网格或 K-Means 聚类将空间划分为簇，目标簇数量约等于目标高斯数
2. 每个簇内的多个高斯椭球合并为一个新高斯
3. 权重基于不透明度和体积计算重要性

**重要**：加权平均不适用于所有参数。控制体积的参数（缩放和旋转）必须用协方差矩阵融合，确保合并后的高斯椭球能填满原来所有高斯椭球共同占据的空间，否则视觉上会出现空洞。

**合并公式**：
- **位置**：加权平均：`x_merged = Σ(w_i * x_i) / Σ(w_i)`
- **不透明度**：加权求和：`opacity_merged = Σ(w_i * opacity_i)` （clamp 到合理范围）
- **缩放 + 旋转**：协方差矩阵融合（**不可简单加权平均**）：
  - 对每个高斯椭球，由 scale 和 rot 构造协方差矩阵 `Σ_i = R_i * diag(exp(scale_i))² * R_i^T`
  - 融合协方差：`Σ_merged = Σ(w_i * (Σ_i + (μ_i - μ_merged)(μ_i - μ_merged)^T))`
    - 第一项：高斯椭球自身的体积散布
    - 第二项：各高斯椭球中心相对于合并后中心的位移散布
  - 对 `Σ_merged` 进行特征分解（Eigendecomposition），提取新的 scale（特征值开方取 log）和 rot（特征向量构成旋转矩阵转四元数）
- **球面调和系数**：加权平均

**权重计算**：`w_i = opacity_i * volume_i`（`volume_i = exp(scale_0_i) * exp(scale_1_i) * exp(scale_2_i)`）

- **优点**：
  - 平滑过渡，不会出现突然的视觉跳变
  - 协方差融合确保合并后高斯覆盖原有全部空间，不会出现空洞
  - 更好地保持场景整体外观
  - 适合大幅度下采样时仍能保持较好视觉质量
- **缺点**：
  - 计算复杂度较高（需要矩阵运算和特征分解）
  - 内存占用较大
  - 可能模糊细节
