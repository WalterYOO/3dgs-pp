# 3DGS 点云处理工具 (3dgs-pp) 产品需求文档

## 1. 项目概述

### 1.1 项目名称
3DGS Point Cloud Processing Tool (3dgs-pp)

### 1.2 项目背景
3D Gaussian Splatting (3DGS) 是一种新型的三维重建技术，其生成的点云数据通常存储在 PLY 文件中。随着场景规模增大，PLY 文件的体积也会急剧增长，给数据查看、处理和传输带来挑战。本工具旨在提供高效的 3DGS PLY 文件处理能力，支持懒加载、终端浏览和空间分块功能。

### 1.3 项目目标
- 提供高效的大型 PLY 文件懒加载机制
- 支持终端交互式浏览高斯点数据
- 实现灵活的空间分块导出功能
- 支持高斯椭球下采样，减少模型复杂度

## 2. 功能需求

### 2.1 懒加载 PLY 文件读取

**功能描述**：支持读取 3DGS 格式的 PLY 文件，采用懒加载方式，避免一次性加载全部数据到内存。

**详细需求**：
- 支持标准 3DGS PLY 文件格式解析
- 仅在需要时加载具体高斯点数据
- 支持文件元数据（顶点数量、属性列表等）的快速读取
- 内存占用控制：元数据加载后内存占用 < 10MB
- 支持超大文件（> 10GB）的处理

**PLY 属性列表**（3DGS 标准）：
- x, y, z：位置坐标
- f_dc_0, f_dc_1, f_dc_2：球面调和基直流分量
- f_rest_0 到 f_rest_44：球面调和基剩余分量（45个）
- opacity：不透明度
- scale_0, scale_1, scale_2：缩放
- rot_0, rot_1, rot_2, rot_3：旋转（四元数）

### 2.2 终端列表展示与翻页

**功能描述**：在终端中以列表形式展示高斯点数据，支持上下翻页浏览。

**详细需求**：

#### 2.2.1 展示模式
- 默认展示核心属性：索引、x/y/z 坐标、opacity
- 支持展开查看更多属性（f_dc、scale、rot）
- 支持自定义选择展示的属性列
- 数值格式化：浮点数保留 6 位小数

**展开模式显示属性：**
- 索引、x、y、z、opacity
- f_dc_0, f_dc_1, f_dc_2
- scale_0, scale_1, scale_2
- rot_0, rot_1, rot_2, rot_3

#### 2.2.2 交互控制
- `j` / `↓`：向下翻一页
- `k` / `↑`：向上翻一页
- `g` / `Home`：跳转到第一页
- `G` / `End`：跳转到最后一页
- `:` / `number` + `Enter`：跳转到指定索引
- `/` + `关键词`：搜索包含特定值的点
- `e`：展开/折叠详细属性
- `q`：退出浏览模式
- `?`：显示帮助信息

#### 2.2.3 页面配置
- 每页默认显示 20 条记录
- 支持通过命令行参数自定义每页条数
- 显示当前位置：`第 X 页 / 共 Y 页 (记录 Z1-Z2 / 共 N)`

### 2.3 空间分块与导出

**功能描述**：按照指定的 X/Y/Z 方向分块数量，将点云分割成多个块，分别导出为 PLY 文件。

**详细需求**：

#### 2.3.1 分块策略
- 输入格式：`Nx*Ny*Nz`，例如 `2*3*2`
- X 方向分 Nx 块，Y 方向分 Ny 块，Z 方向分 Nz 块
- 分块边界：基于点云的包围盒（AABB）均匀划分
- 边界点处理：位于分块边界上的点归属于索引较小的块

#### 2.3.2 导出文件
- 文件命名格式：`{原文件名}_block_{i}_{j}_{k}.ply`
- 每个分块文件包含完整的 PLY 头部信息
- 保留所有原始属性数据
- 生成 `{原文件名}_block_info.json`，记录各分块的：
  - 空间范围（min/max x/y/z）
  - 点数量
  - 文件名映射

#### 2.3.3 进度反馈
- 显示整体进度百分比
- 显示已处理/总点数
- 显示已生成/总分块文件数
- 估算剩余时间

### 2.4 高斯椭球下采样

**功能描述**：对 3DGS 模型进行下采样，按照指定百分比或目标数量保留高斯椭球，生成新的 PLY 文件。

**详细需求**：

#### 2.4.1 采样策略

**V1.0 - 均匀采样（优先实现）**：
- 按索引均匀间隔采样：`保留点索引 = i * step`，其中 `step = 原始点数 / 目标点数`
- 简单高效，适合快速预览
- 适用于点分布相对均匀的场景

**可选高级策略（后续版本）**：

**策略 A - 基于不透明度的重要性采样**：
- 优先保留不透明度高的高斯椭球（对渲染质量影响更大）
- 可配置不透明度权重：`score = opacity * scale_factor`
- 适合需要保持视觉质量的场景

**策略 B - 基于体积的采样**：
- 根据高斯椭球的体积（scale 乘积）进行采样
- 优先保留体积大的高斯椭球
- 适合几何结构保持

**策略 C - 空间聚类采样（Voxel-based）**：
- 将空间划分为体素网格
- 每个体素保留一个代表性高斯椭球（不透明度最高或居中）
- 能够更好地保持空间分布

**策略 D - 随机采样**：
- 完全随机选择保留的点
- 可设置随机种子保证可复现性
- 适合数据增强或统计分析

**策略 E - 高斯椭球合并（Gaussian Merging）**：
- 按空间位置将临近的高斯椭球合并成一个新的高斯椭球
- 不是简单保留/删除，而是通过加权平均整合临近高斯的属性
- 能够更好地保持场景的整体视觉效果
- 适用于需要平滑下采样效果的场景

#### 2.4.2 输入参数

支持两种指定方式（二选一）：
- `--ratio R`：保留百分比，R 为 0 到 1 之间的小数（如 `--ratio 0.5` 表示保留 50%）
- `--count N`：保留的高斯椭球数量，N 为正整数

通用参数：
- `--output FILE`：输出文件路径（默认：`{原文件名}_downsampled.ply`）
- `--method NAME`：采样方法（默认：`uniform`）
  - `uniform`：均匀采样
  - `opacity`：基于不透明度采样（可选）
  - `voxel`：体素聚类采样（可选）
  - `random`：随机采样（可选）
  - `merge`：高斯椭球合并（可选）
- `--seed S`：随机种子（仅随机采样时使用）

#### 2.4.3 输出结果

- 生成新的 PLY 文件，包含下采样后的高斯椭球
- 保留所有原始属性数据
- 在 PLY header 中添加注释记录下采样参数
- 控制台输出统计信息：
  - 原始点数
  - 保留点数
  - 压缩比例
  - 处理时间

#### 2.4.4 进度反馈
- 显示采样进度
- 显示已处理/总点数
- 显示采样方法和参数

### 3.2 兼容性需求

## 4. 命令行接口设计

### 4.1 主命令
```bash
3dgs-pp <command> [options] <ply_file>
```

### 4.2 子命令

#### 4.2.1 info - 查看文件信息
```bash
3dgs-pp info <ply_file>
```
输出：
- 文件路径
- 总点数
- 属性列表
- 包围盒信息（min/max x/y/z）

#### 4.2.2 view - 终端浏览
```bash
3dgs-pp view [--page-size N] [--full] <ply_file>
```
选项：
- `--page-size`：每页显示记录数（默认 20）
- `--full`：默认展开完整属性

#### 4.2.3 split - 空间分块
```bash
3dgs-pp split [--output-dir DIR] <split_spec> <ply_file>
```
参数：
- `split_spec`：分块规格，如 `2*3*2`
- `--output-dir`：输出目录（默认与输入文件同目录）

示例：
```bash
3dgs-pp split 2*3*2 scene.ply
3dgs-pp split --output-dir ./blocks 4*4*4 scene.ply
```

#### 4.2.4 downsample - 下采样
```bash
3dgs-pp downsample [--ratio R | --count N] [--method NAME] [--output FILE] <ply_file>
```

参数：
- `--ratio R`：保留比例，0 < R ≤ 1（如 0.5 表示保留 50%）
- `--count N`：保留的高斯椭球数量
- `--method NAME`：采样方法（默认：uniform）
  - `uniform`：均匀采样
  - `opacity`：基于不透明度采样
  - `voxel`：体素聚类采样
  - `random`：随机采样
  - `merge`：高斯椭球合并
- `--output FILE`：输出文件路径
- `--seed S`：随机种子（仅随机采样）

示例：
```bash
# 按比例下采样（保留 50%）
3dgs-pp downsample --ratio 0.5 scene.ply

# 按数量下采样（保留 10000 个）
3dgs-pp downsample --count 10000 scene.ply

# 指定采样方法和输出文件
3dgs-pp downsample --ratio 0.3 --method opacity --output scene_small.ply scene.ply
```

## 5. 技术架构

### 5.1 核心模块

```
3dgs-pp/
├── ply/
│   ├── __init__.py

│   └── downsampler.py  # 下采样算法
└── main.py
```

### 5.2 关键技术点

1. **懒加载实现**：基于文件偏移量索引，使用 `seek()` 按需读取
2. **终端 UI**：使用 `curses` 或 `rich` 库实现交互式界面
3. **进度条**：使用 `tqdm` 库展示处理进度
4. **下采样算法**：
   - 均匀采样：索引步进算法
   - 重要性采样：基于不透明度/体积的权重排序
   - 体素采样：空间网格划分 + 局部最优选择

### 5.3 下采样算法说明

#### 均匀采样（Uniform Sampling）
**原理**：按固定间隔从原始点云中选取点
- 计算采样步长：`step = original_count / target_count`
- 选取索引：`i * step` 向下取整
- **优点**：O(N) 时间复杂度，实现简单，速度极快
- **缺点**：可能丢失重要特征，不适合点密度变化大的场景

#### 基于不透明度采样（Opacity-based Sampling）
**原理**：优先保留不透明度高的高斯椭球
- 计算每个点的重要性分数：`score = opacity`
- 按分数降序排序，选取前 N 个
- **优点**：保留视觉上更重要的点，渲染质量下降更平缓
- **缺点**：需要 O(N log N) 排序时间，可能破坏空间分布均匀性

#### 体素聚类采样（Voxel-based Sampling）
**原理**：空间划分后每个体素保留代表点
1. 根据点云边界和目标点数计算体素大小
2. 将每个点分配到对应体素
3. 每个体素中选择不透明度最高的点作为代表
- **优点**：保持空间分布均匀性，同时考虑重要性
- **缺点**：内存占用较高（需要构建体素索引），实现较复杂

#### 随机采样（Random Sampling）
**原理**：随机选择 N 个点保留
- 使用 Fisher-Yates shuffle 或 reservoir sampling
- 可设置随机种子保证可复现
- **优点**：无偏采样，适合统计分析
- **缺点**：可能丢失重要特征

#### 高斯椭球合并（Gaussian Merging）
**原理**：将空间临近的高斯椭球合并为一个新的高斯椭球
1. 使用体素网格或 K-Means 聚类将空间划分为簇，目标簇数量约等于目标高斯数
2. 每个簇内的多个高斯椭球合并为一个新高斯
3. 权重基于不透明度和体积计算重要性

**重要**：加权平均不适用于所有参数。控制体积的参数（缩放和旋转）必须用协方差矩阵融合，确保合并后的高斯椭球能填满原来所有高斯椭球共同占据的空间，否则视觉上会出现空洞。

**合并公式**：
- **位置**：加权平均：`x_merged = Σ(w_i * x_i) / Σ(w_i)`
- **不透明度**：加权求和：`opacity_merged = Σ(w_i * opacity_i)` （clamp 到合理范围）
- **缩放 + 旋转**：协方差矩阵融合（**不可简单加权平均**）：
  - 对每个高斯椭球，由 scale 和 rot 构造协方差矩阵 `Σ_i = R_i * diag(exp(scale_i))² * R_i^T`
  - 融合协方差：`Σ_merged = Σ(w_i * (Σ_i + (μ_i - μ_merged)(μ_i - μ_merged)^T))`
    - 第一项：高斯椭球自身的体积散布
    - 第二项：各高斯椭球中心相对于合并后中心的位移散布
  - 对 `Σ_merged` 进行特征分解（Eigendecomposition），提取新的 scale（特征值开方取 log）和 rot（特征向量构成旋转矩阵转四元数）
- **球面调和系数**：加权平均

**权重计算**：`w_i = opacity_i * volume_i`（`volume_i = exp(scale_0_i) * exp(scale_1_i) * exp(scale_2_i)`）

- **优点**：
  - 平滑过渡，不会出现突然的视觉跳变
  - 协方差融合确保合并后高斯覆盖原有全部空间，不会出现空洞
  - 更好地保持场景整体外观
  - 适合大幅度下采样时仍能保持较好视觉质量
- **缺点**：
  - 计算复杂度较高（需要矩阵运算和特征分解）
  - 内存占用较大
  - 可能模糊细节
